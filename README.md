# Top-Down Tactical Command MVP

Браузерный MVP 2D тактического командного прототипа. Игрок управляет не каждым солдатом отдельно, а отделением. Бойцы сами пытаются выполнять приказ, залегают под огнём, подавляются, ищут укрытие и объясняют своё поведение в правой панели.

## Онлайн запуск

Самый простой способ проверить без установки:

```text
https://cdn.jsdelivr.net/gh/AndrewVerhoturov1/external-agent-read-test@main/index.html
```

Если CDN ещё не обновился, подождите немного или откройте файл из GitHub локально через сервер.

## Запуск на Windows

1. Скачайте репозиторий ZIP с GitHub и распакуйте.
2. Откройте папку репозитория.
3. В адресной строке Проводника напишите `cmd` и нажмите Enter.
4. Выполните:

```bash
python -m http.server 8000
```

5. Откройте в браузере:

```text
http://localhost:8000
```

Просто двойной клик по `index.html` может не сработать в некоторых браузерах, потому что сценарий загружается из `data/scenario_01.json`.

## Управление

- `Пуск` / `Пауза` — запустить или остановить симуляцию.
- `Шаг` — один тик симуляции для отладки.
- `Сброс` — начать сценарий заново.
- `Move` — приказ двигаться к точке.
- `Take Cover` — приказ идти к точке и занимать ближайшие укрытия.
- `Suppress` — приказ смотреть/подавлять точку, если цель видна.
- `Face` — повернуть отделение в сектор.
- `Быстро / Обычно / Осторожно` — режим движения.
- Левый клик по бойцу — выбрать бойца.
- Левый клик по карте — отдать приказ выбранному отделению.

## Как проверить главный сценарий

1. Нажмите `Move`.
2. Кликните прямо по дому через открытое поле.
3. Нажмите `Пуск`.
4. Переключите слой `Огонь` или `Подавление`.
5. Смотрите, как вражеский пулемёт простреливает поле: у бойцов растёт suppression, часть залегает, отделение может остановиться.
6. Нажмите `Сброс` и попробуйте вести отделение через лесополосу слева или вдоль укрытий.

## Что сделано

- Карта 1200×800 с полем, лесополосой, дорогой, домом, сараем, окопом, высотой, низиной и стенкой.
- Одно отделение игрока: командир, стрелки, ручной пулемётчик.
- Вражеская огневая группа с пулемётом и стрелками.
- Простые приказы отделению: Move, Take Cover, Suppress, Face.
- Автономное поведение бойцов: движение строем, залегание, подавление, попытка сместиться к укрытию, ответный огонь.
- Упрощённая видимость через здания, лес и рельеф.
- Огонь работает как давление и подавление; модель ранений отключена в текущей версии MVP.
- Правая панель объясняет, почему отделение или боец остановился, залёг или двигается.
- Слои карты: обычный, укрытия, высоты, видимость, огонь, подавление, приказы.
- Журнал событий снизу.

## Честные ограничения MVP

- Это не полноценный варгейм, а маленькая лаборатория поведения.
- Баллистика условная: нет реальной траектории, боеприпасов и точной модели ранений.
- Рельеф влияет упрощённо: высота помогает/мешает видимости только грубой проверкой.
- Окоп считается зоной укрытия вокруг линии, а не точной геометрией траншеи.
- Враг статичен: он не маневрирует, а в основном видит и стреляет.
- Мораль и усталость простые, без сложной психологии.
- Сохранений, редактора карт, кампании, артиллерии и техники нет.

## Следующий логичный шаг

1. Сделать нормальный маршрутизатор движения по укрытиям, а не движение по прямой.
2. Добавить отдельную команду «остановиться / закрепиться».
3. Добавить понятный сектор подавления мышью.
4. Разделить солдат и группы врага на более честную модель обнаружения.
5. Добавить маленький тестовый сценарий без боя для проверки движения и строя.
